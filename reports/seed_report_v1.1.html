근거 문서: seed.yaml · seed_version 1.1 · 추천 개수: 기본 3작 + 옵션 0~2작(합계 3~5작)

1. 사용할 외부 API (무료·저비용 위주)

seed.yaml의 recommended_free_apis와 제약을 반영했습니다. 검색 백엔드는 교체 가능한 인터페이스로 두고, 무료 API를 우선 연동하는 구성을 권장합니다.

API 비용·인증 역할
AniList GraphQL
문서		 무료 공개 쿼리 시 API 키 불필요 장르·태그·점수·제목 등 메타데이터·후보 작품 풀 보강, 추천 근거의 구조화에 유리
Jikan (MyAnimeList 비공식)
문서		 무료 Rate limit 존재 작품 검색·상세 조회로 검색 근거 보강 (AniList와 상호 검증 가능)
DuckDuckGo 검색 (LangChain 커뮤니티 래퍼 등) 무료 API 키 없이 MVP에 자주 사용 웹 검색 도구의 기본 구현 또는 Tavily 폴백
OpenAI API 유료 (프로젝트 제약상 필수) 챗 모델 gpt-5-mini, 필요 시 임베딩 text-embedding-3-small. temperature/top_p 미사용
Tavily 저비용·크레딧 (완전 무료 아님) 인터뷰 선호: LLM 친화 검색. 키가 있으면 DuckDuckGo 대신 또는 병행

우선순위 제안: AniList + Jikan + DuckDuckGo로 비용을 최소화하고, 검색 품질이 부족할 때만 Tavily를 추가합니다.

2. 실제 구현할 Tool 리스트

ReAct 루프에 바인딩할 도구를 역할 단위로 정리했습니다. 최종 추천 개수는 3 + extra_recommendation_slots(0~2)이므로, finalize 단계의 프롬프트·검증에 extra_recommendation_slots를 반드시 전달합니다.

필수·권장 Tool

도구 이름 (예시) 기능 연동 대상
web_search 일반 웹 검색 쿼리 실행, 스니펫·URL 반환 DuckDuckGo (기본) / Tavily (선택)
anilist_search 키워드·장르·연도 등으로 애니 후보 검색, 점수·태그 요약 AniList GraphQL
anime_lookup_jikan 제목으로 MAL 기반 상세(에피소드·장르·점수 등) 조회 Jikan API
finalize_recommendations_ko (그래프 노드) 수집 정보를 바탕으로 정확히 3 + extra_recommendation_slots개의 항목 생성, 한국어 근거 작성, 스포일러 회피 (최종 사용자 대면 텍스트에만 엄격) gpt-5-mini (도구가 아닌 후처리 노드 권장)

ReAct에 직접 노출하는 것은 보통 web_search, anilist_search, anime_lookup_jikan이며, 개수·스포일러 정책은 finalize 노드에서 일괄 적용하는 편이 seed.yaml과 잘 맞습니다.

3. LangGraph 워크플로우 시각화

전체 흐름(입력·추가 슬롯·분기·ReAct·최종 출력)과 ReAct 내부 루프를 표현했습니다. 브라우저에서 Mermaid가 보이지 않으면 mermaid.live에 코드를 붙여 넣어 확인할 수 있습니다.

3.1 상위 그래프 (추가 추천 옵션 + 장르·분위기 분기 + ReAct)

flowchart TB START([시작]) --> UI_OPT[UI: extra_recommendation_slots 선택 0/1/2] UI_OPT --> INGEST[ingest: 폼+자유텍스트 수집] INGEST --> PARSE[parse_profile: 장르·분위기·기타 추출] PARSE --> ROUTER{route_by_genre_mood} ROUTER -->|예: 액션 계열| SG_A[프롬프트/가중치 A] ROUTER -->|예: 일상·힐링| SG_B[프롬프트/가중치 B] ROUTER -->|기본| SG_DEF[프롬프트 기본] SG_A --> REACT[ReAct 에이전트 서브그래프] SG_B --> REACT SG_DEF --> REACT REACT --> FINALIZE["finalize: 3+N작·한국어·스포일러 없음 (N=extra_recommendation_slots)"] FINALIZE --> STREAMLIT[Streamlit: 결과 + 단계 접기/펼치기] STREAMLIT --> END([종료])

3.2 ReAct 서브그래프 (도구 루프)

flowchart LR AGENT[에이전트 노드: gpt-5-mini] --> DECIDE{도구 호출?} DECIDE -->|예| TOOLS[도구 실행: web_search / anilist / jikan] TOOLS --> AGENT DECIDE -->|아니오| OUT[상태에 검색 요약·후보 누적]

3.3 온톨로지 필드 흐름 (ontology_schema)

flowchart TD UP[user_profile] --> BS[branch_state] XS[extra_recommendation_slots] --> FINAL[recommendations 길이 3+N] BS --> TT[tool_trace] TT --> FINAL

4. 그 외 요구사항 정리

4.1 목표·제약 (v1.1 반영)

  • 목표: ReAct + 도구 + 장르·분위기 분기가 있는 LangGraph 워크플로우를 Streamlit으로 제공.
  • 추천 개수: 기본 3작, 사용자 옵션으로 0~2작 추가 → 합계 최소 3, 최대 5. 상태 필드 extra_recommendation_slots ∈ {0,1,2}.
  • LLM: gpt-5-mini 고정. temperature/top_p 등 미사용.
  • UI: 최종 추천이 메인; 도구 호출·중간 단계는 접었다 펼치기.
  • 입력: 하이브리드(간단 폼 + 자유 텍스트).
  • 스포일러: 최종 사용자 대면 추천·근거에서 중요 스포일러 회피. 중간 로그는 상대적으로 완화 가능.
  • 비범위: 방송·판권·지역 라이선스 등 법적 세부 미포함.
  • 검색: 백엔드 교체 가능; 선호는 Tavily 계열.

4.2 비목표 (non_goals)

  • 사용자 계정·로그인·장기 저장(명시적 요구 없음).
  • 파인튜닝·로컬 오픈소스 LLM(명시적 요구 없음).

4.3 수용 기준 (acceptance_criteria)

  • 추천 목록이 최소 3개·최대 5개로 표시된다(기본 3, 옵션 0~2작 추가).
  • 각 추천에 한국어 근거가 있고, 최종 문구에 중요 스포일러가 없다.
  • 그래프에 ReAct(도구 루프)와 장르·분위기 분기가 존재한다.
  • 도구 호출·중간 단계가 접기/펼치기로 확인 가능하다.
  • 웹 검색 또는 동등한 정보 보강이 근거에 반영된다.

4.4 종료 조건 (exit_conditions)

  • recommendations_ready: 3 ≤ recommendations.length ≤ 5, 각 항목에 title, rationale_ko, recommendations.length == 3 + extra_recommendation_slots.
  • user_stop: Streamlit 세션 종료 또는 사용자 취소.
